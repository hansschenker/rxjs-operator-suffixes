## What You Already Know

From Modules 1–9: the complete RxJS operator vocabulary, all four layers, error handling, and resilience patterns. You can build correct, resilient reactive pipelines. This module shows you how to make them readable, testable, and maintainable by a team.

## What This Module Covers

- **10.1** The Alias+Wrap pattern — giving operators domain names with test seams
- **10.2** Hexagonal architecture — isolating RxJS behind domain interfaces
- **10.3** withTelemetry — adding cross-cutting concerns via AOP without modifying operator logic
- **10.4** TestScheduler and marble diagrams — making time-based tests deterministic
- **10.5** The 4-Layer Model as architecture — using the layers as a diagnostic compass

## Why It Matters

Operator knowledge is not enough for production systems. The patterns in this module are what turn a correct pipeline into one that a team can maintain, debug, and extend — without understanding every line of RxJS internals first.

# rxjs-hono Scaffold Design

**Date:** 2026-05-14
**Status:** Approved

## Overview

Full-stack Todos demo app — Hono server with OpenAPIHono + Drizzle + SQLite on the server, RxJS MVU on the client. Single-package layout mirroring `../../marblejs/marblejs-jozef-flakus` so the two projects can be compared side-by-side.

---

## Architecture

Two concurrent processes in dev, one `package.json`:

```
Vite (port 5173)          Hono (port 3000)
RxJS MVU client  ──/api/──▶  OpenAPIHono
fromFetch service            Drizzle + SQLite (todos.db)
custom h() JSX               Scalar interactive docs at /docs
```

Vite proxies `/api/*` → Hono (strips `/api` prefix). The client never references port 3000 directly.

### Server layers (bottom to top)

1. `db/schema.ts` — Drizzle `sqliteTable` definition; `drizzle-zod` derives `TodoSchema`, `CreateTodoSchema`, `UpdateTodoSchema`
2. `todos/todo.routes.ts` — `createRoute()` contracts that reference those Zod schemas
3. `todos/todo.handler.ts` — `AppRouteHandler<typeof route>` implementations; compile error if response shape drifts from contract
4. `app.ts` — `OpenAPIHono` instance; registers all routes; mounts Scalar at `/docs` and OpenAPI JSON at `/openapi.json`
5. `main.ts` — runs `drizzle-kit push` migration then starts Hono on port 3000

### Client layers (identical to Marble.js sibling)

1. `shared/types.ts` — `Todo`, `CreateTodoBody`, `UpdateTodoBody` interfaces
2. `todo.service.ts` — `fromFetch` wrappers returning `Observable<T>`
3. `todo.state.ts` — `action$` Subject → `scan(reducer, initialState)` → `state$` with `shareReplay(1)`
4. `main.tsx` — boot (`getAll$`), `exhaustMap` form submit, `state$.subscribe` render loop

## Data Model

### Drizzle table (`db/schema.ts`)

```typescript
export const todos = sqliteTable('todos', {
  id:        text('id').primaryKey().$defaultFn(() => crypto.randomUUID()),
  title:     text('title').notNull(),
  completed: integer('completed', { mode: 'boolean' }).notNull().default(false),
  createdAt: text('created_at').notNull().$defaultFn(() => new Date().toISOString()),
});

// Derived Zod schemas — single source of truth
export const TodoSchema        = createSelectSchema(todos);
export const CreateTodoSchema  = createInsertSchema(todos).omit({ id: true, createdAt: true });
export const UpdateTodoSchema  = createInsertSchema(todos).partial().omit({ id: true, createdAt: true });
```

### Shared TypeScript types (`shared/types.ts`)

```typescript
export interface Todo           { id: string; title: string; completed: boolean; createdAt: string; }
export interface CreateTodoBody { title: string; }
export interface UpdateTodoBody { title?: string; completed?: boolean; }
```

## Server Patterns

### AppRouteHandler utility type (`server/types.ts`)

```typescript
import type { OpenAPIHono, RouteConfig, RouteHandler } from '@hono/zod-openapi';

export type AppBindings = Record<string, never>;
export type AppOpenAPI   = OpenAPIHono<{ Variables: AppBindings }>;
export type AppRouteHandler<R extends RouteConfig> = RouteHandler<R, { Variables: AppBindings }>;
```

Allows handlers in separate files while preserving full type inference from the route contract.

### Route contract / handler separation

```typescript
// todo.routes.ts
export const listTodosRoute = createRoute({
  method: 'get', path: '/todos',
  responses: {
    200: { content: { 'application/json': { schema: z.array(TodoSchema) } }, description: 'OK' },
  },
});

// todo.handler.ts
export const listTodosHandler: AppRouteHandler<typeof listTodosRoute> = async (c) => {
  const rows = await db.select().from(todos);
  return c.json(rows, 200);
};
```

TypeScript errors if the handler returns a shape or status code not declared in the route contract.

### App assembly (`app.ts`)

```typescript
const app = new OpenAPIHono();
app.openapi(listTodosRoute,   listTodosHandler);
app.openapi(createTodoRoute,  createTodoHandler);
app.openapi(updateTodoRoute,  updateTodoHandler);
app.openapi(deleteTodoRoute,  deleteTodoHandler);
app.doc('/openapi.json', { openapi: '3.0.0', info: { title: 'Todos API', version: '1.0.0' } });
app.get('/docs', apiReference({ spec: { url: '/openapi.json' } }));
```

## Client Patterns

### MVU state (`todo.state.ts`)

```typescript
export const action$ = new Subject<Action>();
export const state$  = action$.pipe(
  scan(reducer, initialState),
  startWith(initialState),
  shareReplay(1),
);
export const dispatch = (action: Action): void => action$.next(action);
```

### Service layer (`todo.service.ts`)

All methods return cold Observables via `fromFetch`. Base URL is `/api/todos` (Vite proxy resolves this).

### Render loop (`main.tsx`)

```typescript
// Boot
getAll$().subscribe({ next: todos => dispatch({ type: 'LOAD_SUCCESS', todos }) });

// Form submit — exhaustMap prevents overlapping requests
fromEvent<SubmitEvent>(form, 'submit').pipe(
  tap(e => e.preventDefault()),
  map(() => titleInput.value.trim()),
  filter(t => t.length > 0),
  exhaustMap(title => create$({ title }).pipe(...)),
).subscribe();

// Render loop
state$.subscribe(({ todos, error }) => { /* re-render list */ });
```

## Testing Strategy

- **Server handlers** — Hono's built-in `.request()` + `testClient` via Vitest; Drizzle uses an isolated in-memory SQLite db per test suite (`:memory:`)
- **Client state** — pure reducer unit tests with `describe/test/expect`
- **Client service** — `fromFetch` wrappers tested with `msw` or by mocking `fetch`
- **h() factory** — DOM tests via `jsdom` environment in Vitest

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/todos` | List all todos |
| POST | `/todos` | Create a todo (`{ title }`) |
| PUT | `/todos/:id` | Update a todo (`{ title?, completed? }`) |
| DELETE | `/todos/:id` | Delete a todo |
| GET | `/docs` | Scalar interactive docs |
| GET | `/openapi.json` | Raw OpenAPI spec |
